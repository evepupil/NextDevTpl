## 职责与边界

本模块只记录 v2.0 的现状、验证依据和后续改造必须保护的行为，不承载新的
产品设计。具体实现以 `v2.0.0` Release 指向的代码为准。

## 结构与数据流

当前应用以 Next.js App Router 为入口，业务能力集中在 `src/features/`，通过
Server Actions、API Routes 和 Inngest 连接数据库及外部服务。

## 关键决策

- 以 `546b53f` 作为 2.x 改造基线。
- 后续重构必须小步提交，并持续保持现有模板可运行。
- v2.0 的完整示例能力可以迁移到 `examples/full/`，不能直接丢失。

## 当前实现

已具备认证、Creem 支付、积分、订阅、后台、工单、存储、邮件、国际化、
内容站点、日志、监控和 AI Provider 接入点。

当前门禁依赖外部 Neon 测试分支。Vitest 使用 `fileParallelism: false` 关闭
测试文件并行，并使用 `sequence.concurrent: false` 关闭文件内用例并发，避免
多组测试同时争用远程数据库连接。
