# 服务适配器

> 模块定位：隔离支付、存储、邮件、AI、任务和限流供应商差异
>
> 对应代码：`src/core/services/`、`src/adapters/`、`src/services/`
>
> 所属里程碑：[M2 - v2.2 服务适配器体系](../roadmap2.x.md#m2)
>
> 当前状态：进行中
>
> 最近更新时间：2026-07-23

## 职责与边界

业务层只使用项目定义的能力接口。供应商 SDK、字段映射、签名校验和错误码
处理全部放在适配器内部。适配器不能改变积分、订阅等核心业务规则，也不能
读取其他业务模块的数据。

M2 覆盖支付、对象存储、事务邮件、文本生成、后台任务和请求限流。页面、API、
Server Action 与业务模块只能调用 `src/services/` 的装配结果或
`src/core/services/` 的稳定类型。

## 结构与数据流

```text
src/core/services/*          稳定契约、能力声明、统一错误
          ^
          | implements
src/adapters/<能力>/<供应商>  SDK 调用、字段转换、签名验证
          ^
          | 完整模板选择默认实现
src/services/*               组合入口
          ^
          | 只暴露项目自己的类型
业务模块、API、Middleware
```

业务用例调用能力接口，生成阶段为每项能力绑定一个实现。外部事件先由支付适配器
转换成内部事件，再交给 webhook 用例处理。内部表结构不保存无必要的供应商
对象。每个适配器显式声明能力差异，例如原生 R2 Binding 不提供 S3 预签名 URL，
Cloudflare Rate Limiting Binding 不返回精确剩余额度。

## 关键决策

- 最终项目只保留所选适配器及其依赖。
- 每类适配器拥有统一契约测试和错误模型。
- webhook 幂等、重试和审计属于公共业务能力。
- OpenAI Compatible 作为 AI 默认接口，特殊能力通过显式扩展暴露。
- 完整模板保持现有默认供应商行为；M3 在生成阶段替换 `src/services/` 的装配
  文件并删除未选适配器。
- Cloudflare 原生绑定通过构造参数注入。M5 生成 `wrangler types` 后将真实
  `Env` binding 传给适配器，M2 不提前引入 Worker 全局类型。
- Cloudflare Email 只用于事务邮件。启用时必须先开通发件域名并配置
  `send_email` binding，同时发送 HTML 和纯文本正文。

## 当前实现

现状审计发现：

- 支付 Action 和 webhook 直接使用 Creem 对象，供应商字段进入 API 业务代码；
- 存储已有 `StorageProvider`，头像 Action 使用该接口，通用上传 API 仍直接创建
  AWS S3 客户端，存在两套实现；
- 邮件发送函数直接获取 Resend 客户端，模板层与发送供应商耦合；
- AI 在单个文件中创建 OpenAI、DeepSeek、MiMo 客户端并读取运行时开关；
- Inngest 路由和示例函数直接引用 SDK，尚无统一的任务派发接口；
- Middleware 和匿名配额工具分别直接创建 Upstash 客户端，存在重复装配；
- 业务源码中还没有 Stripe、R2 Binding、SMTP、Cloudflare Email、Anthropic、
  Workers AI、Workflows 和 Cloudflare Rate Limiting Binding 实现。

## 实施计划

1. 建立六类服务契约、能力声明、错误码和敏感信息清理规则。
2. 迁移现有默认供应商实现，并让业务代码统一通过 `src/services/` 调用。
3. 将支付 webhook 转换为供应商无关内部事件，合并重复的 S3 上传实现。
4. 增加首批扩展适配器，所有外部绑定和 HTTP 客户端均通过构造参数注入。
5. 为每类适配器运行共享契约测试，并增加供应商依赖边界检查。
6. 运行格式化、lint、严格类型检查、完整测试和生产构建。

## 验证方式

- 对每个适配器运行统一契约测试。
- 校验未选择供应商的 SDK、环境变量和源码不会出现在生成结果中。
- 使用注入的测试替身验证 webhook、上传、邮件、AI、任务和限流行为；真实供应商
  冒烟验证随对应部署预设在 M4/M5 完成。
- 检查错误日志中不包含密钥、token 或完整敏感载荷。

## 待扩展项

- M3 根据适配器清单裁剪源码、依赖和环境变量。
- M4 为服务器、Docker Compose 和 Vercel 运行默认适配器真实冒烟验证。
- M5 生成 Cloudflare binding 类型和配置，并运行 R2、Email、Workers AI、
  Workflows、Rate Limiting 的真实部署冒烟验证。
