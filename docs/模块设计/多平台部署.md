# 多平台部署

> 模块定位：支持服务器、Docker Compose 和 Vercel 的正式部署预设
>
> 对应代码：计划覆盖 `deploy/`、`Dockerfile`、`compose.yaml`、部署 recipes
>
> 所属里程碑：[M4 - v2.4 常规部署目标](../roadmap2.x.md#m4)
>
> 当前状态：未开始
>
> 最近更新时间：2026-07-23

## 职责与边界

部署预设负责构建、启动、迁移、健康检查和平台环境变量映射。业务模块不得在
内部判断 Vercel、Docker 或服务器环境，平台差异集中在部署配置和适配器层。

## 结构与数据流

生成器根据部署目标加入对应配置。应用通过统一配置读取数据库、存储、任务和
日志能力；部署预设负责将平台资源映射到这些能力。

## 关键决策

- 服务器自部署使用标准 Node.js 生产产物和进程管理说明。
- Docker Compose 默认使用 named volume 保存数据库数据。
- Compose 中的 Redis、对象存储等服务按模块选择加入。
- Vercel 预设默认组合 Neon、R2、Upstash 和 Inngest。

## 当前实现

当前项目可用于 Vercel 和服务器部署，已有零散说明与构建脚本。Docker
Compose、统一健康检查和部署验证流程尚未形成正式产品能力。

## 验证方式

- 三种目标分别执行生产构建和启动检查。
- 验证数据库迁移、认证回调、webhook、文件上传和后台任务。
- 部署后调用只读健康检查并记录结果。
- 检查容器重启后数据库数据仍然存在。
