# 兼容性与稳定性

> 模块定位：验证模块、适配器和部署目标的受支持组合
>
> 对应代码：计划覆盖 `tests/compatibility/`、CI 工作流和发布检查脚本
>
> 所属里程碑：[M6 - v2.6 兼容性与稳定版](../roadmap2.x.md#m6)
>
> 当前状态：未开始
>
> 最近更新时间：2026-07-23

## 职责与边界

本模块维护官方支持矩阵、自动验证和发布准入条件。它验证已有能力，不承载新
业务功能，也不把所有理论组合都列为官方支持。

## 结构与数据流

CI 调用生成器产出指定组合，依次安装依赖、运行静态检查、测试和构建。涉及
平台的核心组合部署到测试环境，再通过只读冒烟检查确认可访问性。

## 关键决策

- 官方矩阵保持少而明确，新增组合必须承担长期验证成本。
- 核心业务逻辑使用单元与集成测试，纯展示层依赖人工和截图检查。
- Release 必须关联通过的矩阵结果和已知限制。
- 失败日志必须隐藏密钥和用户数据。

## 当前实现

现有 Vitest 测试覆盖多项业务逻辑，尚未建立生成组合、部署目标和适配器契约
的统一验证矩阵。

## 验证方式

- 运行 roadmap 中定义的四组核心组合。
- 对适配器执行契约测试，对部署执行真实冒烟检查。
- 验证升级、卸载模块和替换适配器的文档步骤。
- 检查所有门禁结果与 Release 记录可以追溯。

## 待扩展项

- 确定 CI 预算和外部服务测试账号管理方式。
- 建立失败分类、重试策略和临时豁免流程。
- 设计 2.x 到 3.x 的兼容承诺。
