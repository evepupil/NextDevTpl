# 产品文档总览

NextDevTpl 是一套生产可用的 Next.js SaaS 模板，内置认证、订阅支付、积分账本、管理后台、工单、对象存储、邮件、内容文档、国际化、AI 接入、日志监控和异步任务能力。

这套文档面向产品运营、管理员和开发维护人员，说明每个模块实现了什么、在哪里配置、上线前需要检查什么。

## 推荐阅读路径

1. 先读 [操作手册](/docs/manual)，了解角色、入口和日常流程。
2. 再读 [全局配置](/docs/configuration)，完成环境变量、站点信息、导航和价格配置。
3. 按业务模块阅读 Modules 下的专项说明。
4. 上线前使用 [运营上线检查清单](/docs/operations-checklist) 做最终核对。

## 已覆盖模块

- **认证与账号**：邮箱密码登录、GitHub/Google OAuth、邮箱验证、密码重置、用户角色。
- **数据库与迁移**：PostgreSQL、Drizzle ORM、核心表、迁移命令。
- **订阅支付与积分**：Creem Checkout/Webhook、套餐价格、月度积分、一次性积分包、FIFO 批次和双重记账。
- **用户工作台与管理后台**：仪表盘、设置、用户管理、角色切换、封禁、人工加积分。
- **支持工单**：用户提交工单、追加消息、管理员回复和状态流转。
- **存储与邮件**：S3/R2 兼容存储、头像上传、Resend 和 React Email 模板。
- **内容与国际化**：Fumadocs 文档、博客、法律页面、next-intl 双语路由。
- **AI、可观测性与异步任务**：OpenAI/DeepSeek/MiMo、Upstash 限流、Pino/Axiom、Sentry、Inngest、Cron。

## 快速启动

### 前置条件

- Node.js 20 或更高版本。
- pnpm。
- PostgreSQL 数据库，本地 PostgreSQL 或 Neon 均可。

### 安装

```bash
git clone git@github.com:evepupil/NextDevTpl.git
cd NextDevTpl
pnpm install
cp .env.example .env.local
```

至少填写：

```bash
DATABASE_URL=postgresql://...
BETTER_AUTH_SECRET=your-secret
BETTER_AUTH_URL=http://localhost:3000
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

```bash
pnpm db:push
pnpm dev
```

开发服务默认运行在 `http://localhost:3000`，文档入口是 `/docs`，中文路由下通常是 `/zh/docs`。

## 项目结构

```txt
src/
├── app/                        # Next.js App Router
│   └── [locale]/               # en / zh 国际化路由
├── components/ui/              # Shadcn UI 与 Radix 基础组件
├── features/                   # 业务功能模块
├── db/                         # Drizzle schema
├── lib/                        # Auth、AI、日志、限流等基础能力
├── inngest/                    # 异步任务入口
├── content/                    # docs、blog、legal MDX 内容
└── test/                       # Vitest 测试
```

## 常用命令

```bash
pnpm dev            # 本地开发
pnpm build          # 生产构建
pnpm start          # 启动生产服务
pnpm lint           # Biome lint
pnpm format         # Biome format
pnpm typecheck      # TypeScript 检查
pnpm db:generate    # 生成 Drizzle 迁移
pnpm db:push        # 将 schema 推送到开发库
pnpm db:migrate     # 执行迁移
pnpm db:studio      # 打开 Drizzle Studio
pnpm test           # 运行测试
```
