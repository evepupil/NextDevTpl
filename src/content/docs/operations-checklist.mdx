# 运营上线检查清单

上线前按本清单逐项核对。若某项暂不启用，应记录原因和后续负责人。

## 基础配置

- [ ] `NEXT_PUBLIC_APP_URL` 已改为生产域名。
- [ ] `NEXT_PUBLIC_APP_NAME` 已改为正式产品名。
- [ ] `BETTER_AUTH_SECRET` 是生产强随机密钥。
- [ ] `BETTER_AUTH_URL` 与生产域名一致。
- [ ] `DATABASE_URL` 指向生产数据库。
- [ ] 数据库已完成迁移。
- [ ] `src/config/site.ts` 的产品名、描述、作者、链接已更新。
- [ ] Header、Footer、Dashboard、Admin 导航符合当前产品信息架构。

## 认证与用户

- [ ] 邮箱密码注册可用。
- [ ] 登录、退出、会话刷新可用。
- [ ] 忘记密码流程可用。
- [ ] 邮箱验证邮件可达。
- [ ] GitHub OAuth 回调地址正确。
- [ ] Google OAuth 回调地址正确。
- [ ] 已创建至少一个管理员账号。
- [ ] 管理员无法误改自己的角色。
- [ ] 封禁和解封流程已验证。

## 支付与积分

- [ ] Creem API Key 已配置。
- [ ] Creem Webhook Secret 已配置。
- [ ] Starter、Pro、Ultra 月付和年付价格 ID 已填写。
- [ ] 支付成功跳转 `/dashboard` 正常。
- [ ] 支付取消跳转 `/pricing` 正常。
- [ ] Webhook 能创建或更新 `subscription`。
- [ ] 月度订阅积分发放正确。
- [ ] 注册奖励 200 积分发放正确。
- [ ] FIFO 扣减逻辑通过测试。
- [ ] 管理员人工充值需要原因，且交易记录可查。
- [ ] 重复 Webhook 不会重复发放权益。

## 工作台与管理后台

- [ ] 普通用户能访问 `/dashboard`。
- [ ] 未登录用户访问工作台会被引导登录。
- [ ] 普通用户不能访问 `/admin`。
- [ ] 管理员能访问 `/admin/users` 和 `/admin/tickets`。
- [ ] 用户资料更新后页面刷新正确。
- [ ] 用户搜索、角色调整、封禁、人工充值都可用。
- [ ] 管理员操作有可追踪日志或数据库记录。

## 工单

- [ ] 用户能创建工单。
- [ ] 用户只能查看自己的工单。
- [ ] 用户能在未关闭工单追加消息。
- [ ] 关闭后的工单不能追加消息。
- [ ] 管理员能查看全部工单。
- [ ] 管理员首次回复会将 `open` 转为 `in_progress`。
- [ ] 管理员能将工单改为 `resolved` 或 `closed`。
- [ ] 已定义高优先级工单的响应 SLA。

## 存储与邮件

- [ ] 对象存储 Endpoint、Region、Access Key、Secret 已配置。
- [ ] 头像 bucket 已创建。
- [ ] `NEXT_PUBLIC_AVATARS_BUCKET_NAME` 已配置。
- [ ] 上传 key 必须包含用户 ID。
- [ ] 只允许图片 MIME 类型上传。
- [ ] 用户不能删除他人文件。
- [ ] Resend API Key 已配置。
- [ ] `EMAIL_FROM` 使用已验证域名。
- [ ] 验证邮件和重置密码邮件在主流邮箱可达。

## 内容与 SEO

- [ ] 首页核心卖点、价格和 FAQ 已更新。
- [ ] `/docs` 已能看到本操作手册。
- [ ] 博客中英文内容路径正确。
- [ ] 法律文档已由负责人确认。
- [ ] PSEO 页面 slug 唯一且内容无空字段。
- [ ] `messages/en.json` 和 `messages/zh.json` 文案完整。
- [ ] sitemap 包含关键页面。
- [ ] 页面 title 和 description 已检查。

## AI、任务与监控

- [ ] `AI_PROVIDER` 选择正确。
- [ ] 对应 AI API Key 和模型名可用。
- [ ] 高成本 AI 接口已接入积分扣减和限流。
- [ ] Upstash Redis 已配置，关键 API 限流生效。
- [ ] Axiom 日志已配置或确认使用 console。
- [ ] Sentry DSN 已配置。
- [ ] Inngest 生产密钥已配置。
- [ ] Cron 密钥已配置。
- [ ] 积分过期任务可手动触发并返回合理结果。

## 质量检查

上线前运行：

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

如果某项失败，不建议上线。确需带风险上线，应记录：

- 失败命令。
- 失败原因。
- 影响范围。
- 回滚方案。
- 修复负责人和截止时间。

## 上线后 24 小时监控

| 指标 | 目标 |
| --- | --- |
| 注册成功率 | 无明显异常下降 |
| 登录错误率 | 无集中报错 |
| 支付成功率 | 与 Creem 控制台一致 |
| Webhook 失败数 | 接近 0 |
| 积分余额异常 | 0 起未解释异常 |
| 工单新增 | 高优先级及时响应 |
| Sentry 新错误 | 按严重程度当天处理 |
| AI 成本 | 不超过预算阈值 |
| API 限流 | 无误伤核心用户 |

## 回滚准备

- [ ] 保留上一个可用部署版本。
- [ ] 数据库迁移有回滚方案。
- [ ] 支付 Webhook 可临时暂停或切换。
- [ ] AI Provider 可切换或关闭。
- [ ] 管理员可以冻结异常积分账户。
- [ ] 客服有统一对外说明模板。
