# AI、可观测性与异步任务

该模块负责外部 AI 调用、API 限流、结构化日志、错误监控和后台任务。所有可选服务都设计为可降级，未配置时不应阻塞本地开发。

## 实现了什么

- OpenAI、DeepSeek、MiMo 三种 AI Provider。
- 可选 Cloudflare AI Gateway。
- 通用 Chat Completion 封装。
- Upstash Redis 滑动窗口限流。
- Pino 结构化日志。
- Axiom 云端日志上报。
- Sentry 错误监控。
- Inngest 异步任务。
- Cron 过期积分处理入口。

## AI 相关文件

| 文件 | 作用 |
| --- | --- |
| `src/lib/ai/openai.ts` | Provider 选择、模型选择、客户端和 chatCompletion |
| `src/lib/ai/index.ts` | AI 模块导出 |
| `.env.example` | AI 环境变量说明 |

## AI 配置

```bash
AI_PROVIDER=openai

OPENAI_API_KEY=sk-your-openai-api-key
OPENAI_MODEL=gpt-4o-mini

DEEPSEEK_API_KEY=sk-your-deepseek-api-key
DEEPSEEK_MODEL=deepseek-chat

MIMO_API_KEY=sk-your-mimo-api-key
MIMO_MODEL=mimo-v2-flash
```

`AI_PROVIDER` 可选：

- `openai`
- `deepseek`
- `mimo`

未设置时默认使用 `openai`，默认模型为 `gpt-4o-mini`。

## Cloudflare AI Gateway

如果配置以下变量，OpenAI 或 MiMo 调用会走 Cloudflare AI Gateway：

```bash
CF_AIG_BASE_URL=https://gateway.ai.cloudflare.com/v1/<account>/<gateway>/<provider>/v1
CF_AIG_TOKEN=cf_aig_token_xxx
```

网关 token 会作为 `cf-aig-authorization` 请求头发送。

## AI 调用方式

业务代码可调用：

```ts
import { chatCompletion } from "@/lib/ai";

const result = await chatCompletion(
  [{ role: "user", content: "Generate a summary" }],
  { temperature: 0.7, maxTokens: 1000, jsonMode: false },
);
```

如果开启 `jsonMode`，会设置 OpenAI 兼容的 JSON response format。

## 限流

限流使用 Upstash Redis：

```bash
UPSTASH_REDIS_REST_URL=https://xxx.upstash.io
UPSTASH_REDIS_REST_TOKEN=xxx
```

未配置时限流模块应降级为 pass-through。上线后建议对高成本接口、AI 接口、上传接口和公开 API 配置限流。

## 日志

日志模块使用 Pino，配置 Axiom 后可上报云端：

```bash
AXIOM_TOKEN=xaat-xxx
AXIOM_DATASET=nextdevtpl
```

未配置 Axiom 时输出到控制台。业务事件建议使用统一日志封装，例如支付开始、订阅取消、积分过期等。

## 错误监控

Sentry 配置：

```bash
NEXT_PUBLIC_SENTRY_DSN=https://xxx@xxx.ingest.sentry.io/xxx
SENTRY_AUTH_TOKEN=sntrys_xxx
```

相关文件：

- `sentry.client.config.ts`
- `sentry.server.config.ts`
- `sentry.edge.config.ts`
- `src/instrumentation.ts`
- `src/lib/monitoring/index.ts`

生产环境建议开启 Sentry，并设置 release 和 sourcemap 上传。

## 异步任务

Inngest 相关文件：

| 文件 | 作用 |
| --- | --- |
| `src/inngest/client.ts` | Inngest client |
| `src/inngest/functions.ts` | 后台任务定义 |
| `src/inngest/index.ts` | 模块导出 |
| `src/app/api/inngest/route.ts` | Inngest API 路由 |

开发模式：

```bash
npx inngest-cli@latest dev
pnpm dev
```

可选变量：

```bash
INNGEST_DEV=1
INNGEST_BASE_URL=http://localhost:8288
INNGEST_EVENT_KEY=signkey-prod-xxx
INNGEST_SIGNING_KEY=signkey-prod-xxx
```

## Cron 任务

积分过期处理入口：

```txt
/api/jobs/credits/expire
```

建议配置：

```bash
CRON_SECRET=your-cron-secret
```

调用 Cron 时通过约定 header 或参数校验密钥，避免公开接口被滥用。

## 运营监控指标

| 模块 | 指标 |
| --- | --- |
| AI | 调用量、错误率、平均延迟、Token 成本、Provider 分布 |
| 限流 | 被限流请求数、IP/用户分布、高频接口 |
| 日志 | 支付事件、积分事件、工单事件、管理员操作 |
| Sentry | 错误数、影响用户数、版本分布 |
| Inngest | 任务成功率、重试次数、队列延迟 |
| Cron | 最近运行时间、处理批次数、失败原因 |

## 风险控制

- AI Key、Axiom Token、Sentry Token、Inngest Key 都不能使用 `NEXT_PUBLIC_*`。
- 高成本 AI 接口必须加限流和积分扣减。
- 后台任务必须具备幂等性，避免重试导致重复发放积分。
- Sentry 不应上报明文密钥、密码、支付完整敏感信息。
- Cron 端点必须加密钥保护。
- 生产事故排查优先看 Sentry 错误、Axiom 日志、Webhook 投递日志和数据库交易记录。
